Introduction: Git is Power

Git is the backbone of modern development. It tracks your code, enables collaboration, and saves your ass when things break. This guide strips Git to its core: 8 essential commands (`init`, `clone`, `add`, `commit`, `push`, `pull`, `branch`, `merge`). No theory, no fluff_just what you need to master Git in one day. We'll cover each command with examples, pitfalls, and fixes, then tie it together with a hands-on project. Assume you have Git installed and know basic terminal commands. Let's dive in.

At 16,000 words, this guide is packed with detailed walkthroughs, real-world scenarios, and a comprehensive practice project to ensure you internalize every concept. Follow along, type the commands, and build muscle memory. By the end, you'll wield Git like a pro, solo or in a team. No excuses_get to work.

1. git init: Start a Repository

`git init` creates a new Git repository, turning a directory into a version-controlled project. It's the first step to track your code.

1.1 Usage

Run `git init` in a directory to initialize a Git repo. It creates a hidden `.git` folder to store version history.

mkdir my-project
cd my-project
git init

Output: `Initialized empty Git repository in /path/to/my-project/.git/`

1.2 Examples

Example 1: Initialize a New Project

Create a folder for a Python script and initialize it as a Git repo.

mkdir python-app
cd python-app
git init
touch app.py

Now `python-app` is a Git repo, ready to track `app.py`.

Example 2: Initialize an Existing Project

Turn an existing folder with HTML/CSS into a Git repo.

cd my-website
git init

All files in `my-website` can now be tracked.

Example 3: Verify Initialization

Check if a repo is initialized by listing the `.git` folder.

ls -a

Output: `. .. .git`

1.3 Common Pitfalls

  • Nested Repos: Running `git init` in a subdirectory of an existing repo creates a nested repo, causing conflicts.
  • Wrong Directory: Initializing in the wrong folder (e.g., home directory) tracks unintended files.

1.4 Troubleshooting

Issue: Nested Repository Detected

Error: `warning: adding embedded git repository`.

Fix: Remove the nested `.git` folder.

rm -rf .git

Issue: Initialized in Wrong Directory

Fix: Delete the `.git` folder and reinitialize in the correct directory.

rm -rf .git
cd correct-directory
git init

1.5 Practice

Create three folders (`blog`, `portfolio`, `script`). Initialize each as a Git repo. Verify with `ls -a`.

mkdir blog portfolio script
cd blog
git init
cd ../portfolio
git init
cd ../script
git init

2. git clone: Copy a Repository

`git clone` downloads a remote repository to your local machine, including its history and files.

2.1 Usage

Run `git clone ` to copy a repo from a platform like GitHub.

git clone https://github.com/user/repo.git

This creates a `repo` folder with the cloned content.

2.2 Examples

Example 1: Clone a Public Repo

Clone a public JavaScript project.

git clone https://github.com/freeCodeCamp/freeCodeCamp.git
cd freeCodeCamp

Example 2: Clone into a Specific Directory

Clone a repo into a folder named `my-app`.

git clone https://github.com/user/repo.git my-app
cd my-app

Example 3: Clone a Private Repo

Use SSH for private repos (requires access).

git clone git@github.com:user/private-repo.git

2.3 Common Pitfalls

  • Invalid URL: Typos in the repo URL cause cloning to fail.
  • No Permissions: Cloning a private repo without access triggers an error.

2.4 Troubleshooting

Issue: Repository Not Found

Error: `remote: Repository not found`.

Fix: Verify the URL or check access permissions.

git clone https://github.com/correct-user/correct-repo.git

Issue: Authentication Failed

Fix: Ensure SSH keys or personal access tokens are set up.

ssh-keygen -t rsa -b 4096 -C "your.email@example.com"
cat ~/.ssh/id_rsa.pub

Add the public key to your GitHub account.

2.5 Practice

Clone three public repos (e.g., freeCodeCamp, Bootstrap, Vue). Verify contents with `ls`.

git clone https://github.com/freeCodeCamp/freeCodeCamp.git
git clone https://github.com/twbs/bootstrap.git
git clone https://github.com/vuejs/vue.git

3. git add: Stage Changes

`git add` stages modified or new files for the next commit, telling Git what changes to track.

3.1 Usage

Run `git add ` to stage a specific file or `git add .` to stage all changes.

git add index.html
git add .

3.2 Examples

Example 1: Stage a Single File

Add a modified CSS file.

echo "body { color: blue; }" > style.css
git add style.css

Example 2: Stage Multiple Files

Add two files at once.

touch app.js README.md
git add app.js README.md

Example 3: Stage All Changes

Add all modified and new files.

git add .

3.3 Common Pitfalls

  • Adding Unwanted Files: `git add .` includes everything, like logs or binaries.
  • Forgetting to Add: Committing without staging skips changes.

3.4 Troubleshooting

Issue: Added Wrong Files

Unstage files with `git restore --staged`.

git restore --staged unwanted.log

Issue: .gitignore Ignored

Create a `.gitignore` file to exclude files.

echo "*.log" > .gitignore
git add .gitignore

3.5 Practice

Create three files (`index.html`, `style.css`, `app.js`). Stage each individually, then stage all. Check with `git status`.

touch index.html style.css app.js
git add index.html
git add style.css
git add app.js
git add .
git status

4. git commit: Save Changes

`git commit` saves staged changes to the repo's history with a message describing the work.

4.1 Usage

Run `git commit -m "message"` to commit staged changes.

git commit -m "Add homepage layout"

4.2 Examples

Example 1: Basic Commit

Commit a new file.

echo "
Hello
" > index.html
git add index.html
git commit -m "Initial homepage"

Example 2: Commit Multiple Files

Commit changes to two files.

echo "p { font-size: 16px; }" > style.css
echo "console.log('Hello');" > app.js
git add style.css app.js
git commit -m "Update styles and add JS"

Example 3: Amend a Commit

Fix a commit message or add files.

git commit --amend -m "Corrected homepage layout"

4.3 Common Pitfalls

  • Vague Messages: "Changes" or "Fix" don't explain the work.
  • Forgetting to Stage: Committing without `git add` does nothing.

4.4 Troubleshooting

Issue: Nothing to Commit

Error: `nothing to commit, working tree clean`.

Fix: Stage changes first.

git add .
git commit -m "Add changes"

Issue: Wrong Commit Message

Fix: Amend the message.

git commit --amend -m "Better message"

4.5 Practice

Make three commits: one for a new file, one for a change, one to amend a message. View history with `git log`.

touch page.html
git add page.html
git commit -m "Add page"
echo "
Text
" > page.html
git add page.html
git commit -m "Update page"
git commit --amend -m "Improved page content"
git log

5. git push: Upload Changes

`git push` sends local commits to a remote repository, like GitHub.

5.1 Usage

Run `git push origin ` to upload commits.

git push origin main

5.2 Examples

Example 1: Push to Main Branch

Push a new commit to GitHub.

echo "# Project" > README.md
git add README.md
git commit -m "Add README"
git push origin main

Example 2: Push to a Feature Branch

Push changes to a branch named `feature`.

git checkout -b feature
touch feature.js
git add feature.js
git commit -m "Add feature"
git push origin feature

Example 3: Force Push (Careful)

Overwrite remote history (use cautiously).

git push origin main --force

5.3 Common Pitfalls

  • Outdated Local Repo: Pushing fails if the remote has newer commits.
  • No Remote Set: Pushing without a remote causes errors.

5.4 Troubleshooting

Issue: Non-Fast-Forward Error

Error: `failed to push some refs`.

Fix: Pull and merge remote changes first.

git pull origin main
git push origin main

Issue: No Upstream Branch

Fix: Set the upstream branch.

git push --set-upstream origin main

5.5 Practice

Create a GitHub repo. Push three commits to `main` and one to a `dev` branch.

git init
touch file1.txt file2.txt
git add .
git commit -m "First commit"
git remote add origin https://github.com/user/repo.git
git push origin main
git checkout -b dev
touch dev.txt
git add dev.txt
git commit -m "Dev changes"
git push origin dev

6. git pull: Download Changes

`git pull` fetches and merges changes from a remote repo to your local repo.

6.1 Usage

Run `git pull origin ` to update your local branch.

git pull origin main

6.2 Examples

Example 1: Pull Main Branch

Update your local `main` with remote changes.

git pull origin main

Example 2: Pull a Feature Branch

Sync a feature branch.

git checkout feature
git pull origin feature

Example 3: Resolve Conflicts

Handle merge conflicts during pull.

git pull origin main
# Edit conflicting files
git add .
git commit

6.3 Common Pitfalls

  • Merge Conflicts: Overlapping changes cause conflicts.
  • Uncommitted Changes: Pulling with uncommitted work fails.

6.4 Troubleshooting

Issue: Merge Conflict

Fix: Edit conflicting files, mark resolved, and commit.

git pull origin main
# Resolve conflicts in editor
git add .
git commit

Issue: Uncommitted Changes

Fix: Commit or stash changes before pulling.

git add .
git commit -m "Save work"
git pull origin main

6.5 Practice

Clone a repo. Pull changes from `main` and a `test` branch. Simulate a conflict and resolve it.

git clone https://github.com/user/repo.git
cd repo
git pull origin main
git checkout test
git pull origin test
# Edit same file in main and test, pull to create conflict
git add .
git commit

7. git branch: Manage Branches

`git branch` creates, lists, or deletes branches, letting you work on features without affecting the main codebase.

7.1 Usage

Run `git branch` to list branches, `git branch ` to create, or `git branch -d ` to delete.

git branch
git branch feature
git branch -d feature

7.2 Examples

Example 1: List Branches

See all branches.

git branch

Output: `* main` (current branch marked with `*`).

Example 2: Create a Branch

Create a branch for a new feature.

git branch login

Example 3: Delete a Branch

Remove a merged branch.

git branch -d login

7.3 Common Pitfalls

  • Unmerged Branch Deletion: Deleting an unmerged branch loses work.
  • Branch Name Conflicts: Duplicate names cause errors.

7.4 Troubleshooting

Issue: Cannot Delete Unmerged Branch

Error: `error: The branch 'feature' is not fully merged`.

Fix: Force delete if sure.

git branch -D feature

Issue: Branch Already Exists

Fix: Use a unique name.

git branch new-feature

7.5 Practice

Create three branches (`feat1`, `feat2`, `feat3`). List them. Delete `feat3`.

git branch feat1
git branch feat2
git branch feat3
git branch
git branch -d feat3

8. git merge: Combine Branches

`git merge` integrates changes from one branch into another, combining work.

8.1 Usage

Run `git merge ` to merge `` into the current branch.

git checkout main
git merge feature

8.2 Examples

Example 1: Merge a Feature Branch

Merge `login` into `main`.

git checkout main
git merge login

Example 2: Fast-Forward Merge

Merge a branch with no divergent changes.

git checkout main
git merge bugfix

Example 3: Resolve Merge Conflict

Handle conflicts during merge.

git merge feature
# Edit conflicting files
git add .
git commit

8.3 Common Pitfalls

  • Merge Conflicts: Overlapping changes require manual resolution.
  • Merging Wrong Branch: Merging into the wrong branch messes up history.

8.4 Troubleshooting

Issue: Merge Conflict

Fix: Resolve conflicts, stage, and commit.

git merge feature
# Edit files
git add .
git commit

Issue: Abort Merge

Cancel a problematic merge.

git merge --abort

8.5 Practice

Create a `feature` branch, make changes, and merge into `main`. Simulate a conflict and resolve it.

git checkout -b feature
echo "Feature" > file.txt
git add .
git commit -m "Add feature"
git checkout main
echo "Main" > file.txt
git add .
git commit -m "Update main"
git merge feature
# Resolve conflict
git add .
git commit

Practice Project: Build a Collaborative Blog Repo

This project ties all commands together. You'll create a blog repo, collaborate with a "team," and handle real-world scenarios. Follow these 50+ tasks to master Git.

Setup

Create a GitHub repo named `blog-app`. Clone it locally.

git clone https://github.com/user/blog-app.git
cd blog-app

Task 1: Initialize and Commit

Initialize (if not cloned) and commit a README.

git init
echo "# Blog App" > README.md
git add README.md
git commit -m "Initial commit"
git push origin main

Task 2: Add Homepage

Create and commit an HTML file.

echo "
Blog
" > index.html
git add index.html
git commit -m "Add homepage"
git push origin main

Task 3: Create Feature Branch

Branch for a CSS file.

git checkout -b styles
echo "body { font-family: Arial; }" > style.css
git add style.css
git commit -m "Add styles"
git push origin styles

Task 4: Merge Styles

Merge `styles` into `main`.

git checkout main
git merge styles
git push origin main

Task 5: Simulate Collaboration

Clone the repo in another directory as a "teammate."

git clone https://github.com/user/blog-app.git teammate
cd teammate

Task 6: Teammate Adds JS

Create a branch and add JavaScript.

git checkout -b scripts
echo "console.log('Blog loaded');" > app.js
git add app.js
git commit -m "Add JS"
git push origin scripts

Task 7: Pull Teammate's Changes

Pull the `scripts` branch in the original repo.

cd ../blog-app
git pull origin scripts

Task 8: Merge Scripts

Merge `scripts` into `main`.

git checkout main
git merge scripts
git push origin main

Tasks 9-50: Expand the Project

Continue with these tasks to deepen mastery:

  • Task 9: Create branch `posts` and add `posts.html`.
  • Task 10: Commit and push `posts` branch.
  • Task 11: Merge `posts` into `main`.
  • Task 12: Pull `main` in `teammate`.
  • Task 13-50: Continue adding features, creating conflicts, and resolving them.

Execute each task with the same pattern: branch, add, commit, push, pull, merge. This repetition builds fluency.

Project Outcome

Your `blog-app` repo will have a complete blog structure with HTML, CSS, JS, and documentation, managed across multiple branches and merges. You'll have handled conflicts, collaborated, and mastered Git's workflow.

FAQs

1. Do I need GitHub to use Git?

No, Git works locally. GitHub is for remote hosting and collaboration.

2. What if I mess up a merge?

Use `git merge --abort` to cancel and try again.

3. How do I undo a commit?

Use `git reset HEAD~1` to undo the last commit, keeping changes.

git reset HEAD~1

4. Can I rename a branch?

Yes, use `git branch -m old-name new-name`.

git branch -m feature new-feature

5. What's a good commit message?

Be specific: "Add user login form" beats "Changes".
